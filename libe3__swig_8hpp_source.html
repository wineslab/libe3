1/* SWIG-friendly re-declarations of the libe3 public API.
2 *
3 * SWIG 4.1 has trouble parsing some modern C++ constructs in the real
4 * headers (e.g. brace-initialised default arguments, std::function
5 * typedefs, std::variant). This file mirrors the slice of the API we
6 * want to expose to Python with SWIG-compatible declarations only.
7 *
8 * The actual implementations live in libe3 — this header exists purely
9 * so SWIG can parse simple declarations without needing C++-aware
10 * type recovery.
11 *
12 * SPDX-License-Identifier: Apache-2.0
13 */
