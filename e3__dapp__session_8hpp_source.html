1/**
2 * @file e3_dapp_session.hpp
3 * @brief Python-facing dApp session over libe3 — batched, low-latency seam.
4 *
5 * @section why Why this class exists
6 * libe3's core @ref libe3::E3Agent drives the dApp role with `std::function`
7 * handlers (@ref libe3::E3Agent::set_indication_handler et al.). SWIG cannot
8 * bridge those callbacks to Python across libe3's inbound worker threads
9 * without either running Python under the GIL *on the hot thread* (directors —
10 * deadlock-prone, serialises dispatch) or paying a GIL round-trip per message.
11 * Both are unacceptable at the E3AP target rate (real-time, sub-millisecond
12 * cadence, very high message throughput).
13 *
14 * @ref libe3::py::DAppSession is the answer: it owns an @ref libe3::E3Agent in
15 * DAPP role, registers the C++ handlers itself, and funnels every inbound
16 * message into a **bounded lock-free ring** (@ref libe3::LockFreeQueue). Python
17 * drains that ring in **batches** via @ref poll_events, so one GIL acquire and
18 * one Python↔C++ boundary crossing are amortised across many messages. The
19 * SWIG wrapper releases the GIL around the blocking calls (see swig/libe3.i,
20 * `%module(threads="1")`), so libe3's C++ threads never stall on Python.
21 *
22 * The E3SM (service-model) payloads stay opaque here (`std::vector<uint8_t>`);
23 * dApp-library encodes/decodes them in Python. This class is E3AP only, mirroring
24 * the OAI split where libe3 owns E3AP and the SM owns encode/decode.
25 *
26 * @section lifecycle Lifecycle
27 * ctor (registers handlers) → @ref start → @ref wait_for_setup →
28 * (read setup via @ref dapp_id / setup accessors) → @ref subscribe →
29 * loop: @ref poll_events + @ref send_control / @ref send_report →
30 * @ref release → @ref stop.
31 *
32 * @section rejected Rejected alternatives (auditable)
33 * - SWIG directors: run the Python callback on libe3's inbound thread under the
34 * GIL — serialises dispatch, deadlock-prone if Python re-enters libe3.
35 * - per-message poll: one GIL round-trip + one boundary crossing per message
36 * caps throughput; batching is strictly better with no added latency (the
37 * ring still wakes on the first event).
38 *
39 * SPDX-License-Identifier: Apache-2.0
40 */

58enum E3EventKind {

67/**
68 * @brief One inbound E3AP event, POD-flat for a clean SWIG crossing.
69 *
70 * `payload` carries the opaque E3SM bytes for indications
71 * (protocol_data) and xApp controls (xapp_control_data); it is empty for
72 * subscription/setup responses and acks. Fields not relevant to a given
73 * `kind` are left zero.
74 */
75struct E3Event {
76 int kind{E3_EVENT_NONE}; ///< one of @ref E3EventKind
77 uint32_t dapp_id{0}; ///< dApp identifier the message targets
78 uint32_t ran_function_id{0}; ///< RAN function (indication / xApp control)
79 uint32_t subscription_id{0}; ///< subscription id (subscription response)
80 uint32_t request_id{0}; ///< request id (subscription response / ack)
81 int response_code{-1}; ///< 0=positive, 1=negative, -1=n/a
82 std::vector<uint8_t> payload; ///< opaque E3SM bytes (indication / xApp control)
83 // NOTE: in Python (libe3py) the payload is read via ev.get_payload(), which
84 // returns native `bytes`. SWIG member getters return a wrapped vector, so the
85 // binding exposes the payload as a by-value method instead (see swig/libe3.i).
86};

88/**
89 * @brief dApp-role E3AP session with a batched, lock-free inbound ring.
90 *
91 * Thread model: two producers push into the ring — the setup-response handler
92 * (libe3 setup thread) and the other handlers (libe3 inbound thread), which run
93 * concurrently at startup; the Python drainer thread is the single consumer via
94 * @ref poll_events. The ring is MPMC, so this is safe (do not "optimize" it to
95 * SPSC). All outbound verbs are thread-safe (delegated to @ref libe3::E3Agent).
96 */
97class DAppSession {
98public:
99 /**
100 * @brief Construct the session and register all inbound handlers.
101 * @param config Agent config; `role` is forced to @ref E3Role::DAPP.
102 * @param queue_capacity Inbound ring capacity (rounded up to a power of
103 * two). Sized for high-throughput headroom; overflow increments the
104 * drop counter (@ref dropped_events) rather than blocking libe3.
105 */
106 explicit DAppSession(libe3::E3Config config, std::size_t queue_capacity = 8192);
107 ~DAppSession();
108
109 DAppSession(const DAppSession&) = delete;
110 DAppSession& operator=(const DAppSession&) = delete;
111
112 /** @brief Start the agent, re-arming the inbound ring so a stop()->start()
113 * cycle works again. @return ErrorCode as int. */
114 int start();
115 /** @brief Block until the setup handshake completes. @return ErrorCode as int. */
116 int wait_for_setup(int timeout_ms);
117 /** @brief Send a ReleaseMessage (session keeps running until @ref stop). @return ErrorCode as int. */
118 int release();
119 /** @brief Stop the agent and shut down the ring so a blocked @ref poll_events
120 * returns an empty batch (i.e. empty-after-stop means "done"). Restartable

122 void stop();

164 /**
165 * @brief Drain up to @p max_batch inbound events, waiting up to
166 * @p timeout_ms for the first.
167 *
168 * Blocks on the ring for the first event (latency ≈ ring wakeup), then
169 * sweeps up any further already-queued events without blocking.
170 * @p timeout_ms <= 0 polls non-blocking (does not busy-wait). Returns an
171 * empty vector on a quiet tick or after @ref stop. The SWIG wrapper releases
172 * the GIL for the whole call so libe3's threads run freely while Python waits.
173 */
174 std::vector<E3Event> poll_events(std::size_t max_batch, int timeout_ms);
175
176 /** @brief Total inbound events dropped due to ring overflow (backpressure). */
177 unsigned long long dropped_events() const;
